# CopyTrade EA（MT4）

MT4用のエキスパートアドバイザです。**ファイルキュー**で、1つの口座（送信側）のトレードを別口座（受信側）にコピーします。ターミナル間のネットワークやブローカーAPIは不要で、両方のMT4が**同じCommonフォルダ**を参照する必要があります（同一PCまたは共有ドライブ）。

- **反転トレード:** 送信側BUY → 受信側SELL、送信側SELL → 受信側BUY。
- **複数送信・1受信:** 1つの受信口座で複数の送信口座をフォロー可能。
- **重複防止:** 1つの送信エントリーに対して受信側のポジションは最大1つまで。

```

├── Experts/
│   ├── CopyTrade_Sender.mq4   ← コピー元口座のチャートに設置
│   └── CopyTrade_Receiver.mq4 ← コピー先口座のチャートに設置
├── Include/
│   ├── CT_Event.mqh
│   ├── CT_Ledger.mqh
│   ├── CT_Map.mqh
│   └── CT_Symbol.mqh

```

`Experts/*.mq4` と `Include/*.mqh` をMT4の `MQL4` フォルダにコピーし、インクルードが参照できるようにしてください。

## 送信側（CopyTrade_Sender）

**コピー元**口座で動作します。建玉・決済を監視し、OPEN/CLOSEイベントをキュー用フォルダに書き出します。受信側が開いた注文（マジック番号およびコメント `SRC:`）は無視します。


| パラメータ                   | 説明                                                                                         |
| ----------------------- | ------------------------------------------------------------------------------------------ |
| **ScanIntervalSeconds** | スキャン間隔（秒）。既定値 1。                                                                           |
| **ReceiverMagic**       | 受信側のマジック番号と一致させる（コピー注文を無視するため）。既定値 900001。                                                 |
| **QueueFolder**         | Common配下のイベント用フォルダ名。既定値 `ct_queue`。                                                        |
| **SymbolDelimiter**     | チャート銘柄に接尾語がある場合（例: `XAUUSD-cd`）、区切り文字（例: `-`）を指定。イベントにはベース銘柄名が送信されます。接尾語がない場合は空欄。          |
| **SymbolMapping**       | 銘柄変換（例: `GOLD=XAUUSD`）。複数はカンマ区切り: `GOLD=XAUUSD,SILVER=XAGUSD`。**送信側か受信側のどちらか一方のみで設定すること。** |


---

## 受信側（CopyTrade_Receiver）

**コピー先**口座で動作します。キューからイベントファイルを読み、注文のオープン・クローズを実行します。台帳とマップで同一イベントの二重実行を防ぎ、CLOSE時に正しいポジションを特定します。


| パラメータ             | 説明                                                                        |
| ----------------- | ------------------------------------------------------------------------- |
| **TimerSeconds**  | キュー確認間隔（秒）。既定値 1。                                                         |
| **Slippage**      | OrderSend/OrderClose の最大スリッページ。既定値 10。                                    |
| **ReceiverMagic** | コピー注文のマジック番号。既定値 900001。送信側の設定と一致させる。                                     |
| **LotMultiplier** | ロット倍率（受信ロット = 送信ロット × この値）。既定値 1.0。                                       |
| **QueueFolder**   | 送信側と同じフォルダ名。既定値 `ct_queue`。                                               |
| **SymbolMapping** | イベントの銘柄を受信側銘柄に変換（例: `XAUUSD=GOLD`）。複数はカンマ区切り。**送信側か受信側のどちらか一方のみで設定すること。** |


受信側の注文コメントは `SRC:<送信口座ログイン>|T:<送信側チケット>` となり、どの送信エントリーか識別できます。

## 銘柄の区切り文字と変換

### 区切り文字（送信側）

ブローカーによっては `XAUUSD-cd` や `USDJPY.oj5k` のように、ベース銘柄＋区切り文字＋接尾語になっている場合があります。**SymbolDelimiter** を設定すると、イベントには**ベース銘柄**が送信されます。

- `XAUUSD-cd` → 区切り文字 `-` → イベント銘柄 `XAUUSD`
- `USDJPY.oj5k` → 区切り文字 `.` → イベント銘柄 `USDJPY`
- `XAUUSD`（接尾語なし）→ SymbolDelimiter は**空欄**

### 銘柄変換（どちらか一方のみ）

**SymbolMapping** は**送信側か受信側のどちらか一方**で設定してください。両方では設定しないでください。

- **送信側:** 「自口座の銘柄」→「イベントに書く銘柄」  
例: 送信側チャートが `GOLD`、受信側ブローカーが `XAUUSD` の場合 → **送信側**で SymbolMapping に `GOLD=XAUUSD` を設定。
- **受信側:** 「イベントの銘柄」→「自口座の銘柄」  
例: イベントが `XAUUSD`、受信側チャートが `GOLD` の場合 → **受信側**で SymbolMapping に `XAUUSD=GOLD` を設定。

形式: `KEY=VALUE`。複数: `GOLD=XAUUSD,SILVER=XAGUSD`。

受信側ブローカーが接尾語付き銘柄（例: `XAUUSD.oj5k`）を使う場合は、受信側で `XAUUSD=XAUUSD.oj5k` のように設定してください。

## 動作条件

- 両方のターミナルがキュー（および台帳・マップ）用に**同じCommonパス**を参照していること。同一PCまたは共有ドライブが一般的です。
- 送信側と受信側で **QueueFolder** と **ReceiverMagic** を同じ値にすること。
- ブローカー間で銘柄名が異なる場合は、**SymbolMapping** をどちらか一方で設定し、送信側銘柄に接尾語がある場合は送信側で **SymbolDelimiter** も設定してください。

## データファイル（Commonパス）

- **ct_queue/** — イベントファイル（`.evt`）。送信側が作成し、受信側が読み取り後に削除。
- **ct_ledger/** — 処理済みイベントID（二重実行防止用）。
- **ct_map/** — 送信側（ログイン・チケット）→ 受信側チケット（CLOSE用）。

EA稼働中は、継続運用に必要な限りこれらのフォルダ・ファイルを削除しないでください。
